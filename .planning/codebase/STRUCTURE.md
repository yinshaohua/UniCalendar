## Directory Purposes

**src/:**
- Purpose: TypeScript source code for the plugin
- Contains: Plugin class, settings interface, modal, and command implementations
- Key files: `main.ts` (plugin lifecycle), `settings.ts` (configuration UI)

**.github/:**
- Purpose: GitHub Actions CI/CD workflows
- Contains: Automated linting and release processes
- Key files: Workflow YAML files for lint checks on every commit

**.planning/:**
- Purpose: GSD (Get Stuff Done) planning documents
- Contains: Architecture analysis, structure maps, conventions, and concerns documentation
- Key files: `ARCHITECTURE.md`, `STRUCTURE.md`, `CONVENTIONS.md`, etc.

## Key File Locations

**Entry Points:**
- `src/main.ts`: TypeScript source entry point; compiled to `main.js` by esbuild
- `main.js`: JavaScript bundle loaded by Obsidian; must be at root level of plugin folder

**Configuration:**
- `manifest.json`: Declares plugin metadata (id, name, version, minAppVersion, description, author)
- `tsconfig.json`: TypeScript compiler options (strict mode, ES6 target, DOM + ES7 libs)
- `eslint.config.mts`: ESLint configuration using flat config with Obsidian plugin rules
- `esbuild.config.mjs`: Build bundler config (entry point, external dependencies, output format)
- `package.json`: NPM scripts and dependencies (obsidian, typescript, esbuild, eslint, etc.)

**Core Logic:**
- `src/main.ts`: Plugin lifecycle (onload, onunload), ribbon icon, commands, status bar, global events
- `src/settings.ts`: Settings interface, defaults, and settings tab UI class
- `styles.css`: Optional plugin styles (currently minimal)

**Build Artifacts:**
- `main.js`: Compiled and bundled plugin (generated by esbuild; must be committed for distribution)
- `versions.json`: Maps plugin versions to minimum Obsidian app versions

## Naming Conventions

**Files:**
- Plugin entry: `main.ts` → `main.js` (required by Obsidian)
- Configuration modules: `settings.ts` (interface + defaults)
- Modals/UI: Named descriptively (e.g., `SampleModal`)
- Commands: Grouped in functions or separate modules (recommended pattern: `commands/command1.ts`)
- Build tools: `.mjs` extension for ES modules (e.g., `esbuild.config.mjs`, `version-bump.mjs`)
- Config files: Standard names (`tsconfig.json`, `manifest.json`, `eslint.config.mts`)

**Directories:**
- Source code: `src/` (lowercase)
- Commands (recommended): `src/commands/` (plural)
- UI components: `src/ui/` (lowercase)
- Utilities: `src/utils/` (lowercase)
- Types: Co-located with usage or in `src/types.ts`

## Where to Add New Code

**New Feature:**
- Primary code: `src/main.ts` (register command via `this.addCommand()`)
- Feature logic: Extract to `src/commands/{feature-name}.ts`
- Tests: `src/{feature-name}.test.ts` (co-located with source)

**New Component/Modal:**
- Implementation: `src/ui/{component-name}.ts` (extends Obsidian Modal or View)
- Instance creation: Register in `src/main.ts` within `onload()`

**New Setting:**
- Interface: Update `MyPluginSettings` interface in `src/settings.ts`
- Default: Add to `DEFAULT_SETTINGS` object in `src/settings.ts`
- UI: Add `new Setting()` call in `SampleSettingTab.display()` method

**Utilities:**
- Shared helpers: `src/utils/helpers.ts`
- Constants: `src/utils/constants.ts` or `src/constants.ts`
- Types: `src/types.ts` or co-locate with usage

**Global Listeners/Events:**
- Register: In `onload()` method of `src/main.ts` using `this.register*()` helpers
- Logic: Extract to separate functions and import into `src/main.ts`

## Special Directories

**node_modules/:**
- Purpose: NPM dependencies
- Generated: Yes (via `npm install`)
- Committed: No (in `.gitignore`)

**main.js (root level):**
- Purpose: Compiled plugin bundle required by Obsidian
- Generated: Yes (by esbuild via `npm run build` or `npm run dev`)
- Committed: Yes (required for distribution via GitHub releases)

**.obsidian/plugins/{plugin-id}/ (runtime, not in this repo):**
- Where Obsidian installs plugins in a vault
- Contains symlink to dev folder or copied `main.js`, `manifest.json`, `styles.css`
