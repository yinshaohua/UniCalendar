# Phase 6 — UI Design Contract

> Visual and interaction contract for Chinese Lunar Calendar Support. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Primary focal point: today date number circle (accent background).**

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Obsidian plugin -- no React/shadcn) |
| Preset | not applicable |
| Component library | Obsidian API (Modal, ItemView, Notice, Setting, setIcon) |
| Icon library | Lucide (bundled with Obsidian, accessed via `setIcon()`) |
| Font | `var(--font-interface)` (Obsidian system font stack) |

**Note:** Inherits all design system decisions from Phase 2 UI-SPEC. Phase 5 Apple Calendar polish (color-mix patterns, soft shadows, rounded corners) is the current visual baseline. All new elements must match that aesthetic.

---

## Spacing Scale

Inherited from Phase 2. No new tokens. Declared values (Obsidian `--size-4-*` tokens):

| Token | Value | Obsidian Variable | Usage |
|-------|-------|-------------------|-------|
| xs | 4px | `var(--size-4-1)` | Icon-to-text gap, lunar text inline spacing |
| sm | 8px | `var(--size-4-2)` | Badge internal padding, compact element gaps |
| md | 12px | `var(--size-4-3)` | View padding, toolbar margin-bottom |
| lg | 16px | `var(--size-4-4)` | Settings section gaps |
| xl | 24px | `var(--size-4-6)` | Not used in this phase |

**Exception -- md token (12px):** 12px is not in the canonical 8-point set {4, 8, 16, 24, 32, 48, 64}. It is a justified exception: Obsidian's own `--size-4-3` maps to 12px, and this compact plugin view density requires an intermediate step between 8px and 16px. We adopt Obsidian's native token rather than forcing a canonical value that would feel too loose (16px) or too tight (8px) for view padding and toolbar margins.

Exceptions:
- Holiday badge ("xiu"/"ban"): `padding: 0 4px` -- 0px vertical, 4px horizontal for compact corner badge while maintaining 4px-grid alignment
- Lunar text next to day number: `margin-left: 4px` inline gap
- Month title lunar month text: `margin-left: 8px` gap from Gregorian title

## Typography

Inherits Phase 2 baseline. New roles added for lunar calendar elements:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | `var(--font-ui-small)` (~13px) | 400 | 1.4 | Event bar text, modal body text |
| Label | `var(--font-ui-smaller)` (~11px) | 400 | 1.3 | Hour labels, overflow indicator |
| Heading | `var(--font-ui-small)` (~13px) | 600 | 1.2 | Toolbar month/date label |
| Lunar date | `var(--font-ui-smaller)` (~11px) | 400 | 1.3 | Lunar day text next to Gregorian number in month view |
| Lunar festival | `var(--font-ui-smaller)` (~11px) | 600 | 1.3 | Traditional festival names (semibold to distinguish from regular lunar dates) |
| Solar term | `var(--font-ui-smaller)` (~11px) | 400 | 1.3 | Solar term names in month view (regular weight, distinguished by color only) |
| Lunar month (title) | `var(--font-ui-small)` (~13px) | 400 | 1.2 | Lunar month name appended to toolbar title |
| Holiday badge | `var(--font-ui-smaller)` (~11px) | 600 | 1.0 | "xiu"/"ban" corner badge text |

**Sizes used (4 tokens):** `var(--font-ui-smaller)` (~11px), `var(--font-ui-small)` (~13px). Two Obsidian tokens plus inherited Body and Heading sizes from Phase 2. Total distinct size expressions: `var(--font-ui-smaller)`, `var(--font-ui-small)`, plus Phase 2 inherited body (`var(--font-ui-medium)`) and large heading sizes. This phase declares only 2 new size tokens.

**Weights used (2):** 400 (body, lunar date, solar term, lunar month title) and 600 (heading, festival, badge). Solar terms rely on color for distinction -- weight difference is not needed. Headings use 600 semibold instead of 700 bold for consistency with the two-weight constraint.

## Color

Inherits Phase 2/5 baseline. New semantic colors added for lunar calendar layer:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background-primary)` | Grid cells, view background |
| Secondary (30%) | `var(--background-secondary)` | Day headers, hour labels |
| Accent (10%) | `var(--interactive-accent)` | Today date number circle (background), active view tab |
| Destructive | `var(--text-error)` | Sync error text only |
| Solar term text | `color-mix(in srgb, var(--interactive-accent) 80%, var(--text-normal))` | Solar term names -- accent-tinted for visual distinction from plain lunar dates |
| Lunar date text | `var(--text-faint)` | Regular lunar day numbers -- subdued to not compete with Gregorian date |
| Lunar festival text | `var(--text-accent)` | Traditional festival names -- uses Obsidian accent text color |
| Holiday rest background | `color-mix(in srgb, #ef4444 8%, var(--background-primary))` | Statutory holiday day cell background -- very light warm tint |
| Holiday rest badge bg | `color-mix(in srgb, #ef4444 75%, var(--background-primary))` | "xiu" badge background -- visible red |
| Holiday rest badge text | `#ffffff` | "xiu" badge text -- white on red |
| Workday adjust background | `color-mix(in srgb, #f59e0b 8%, var(--background-primary))` | Adjusted workday cell background -- very light amber tint |
| Workday adjust badge bg | `color-mix(in srgb, #f59e0b 75%, var(--background-primary))` | "ban" badge background -- visible amber |
| Workday adjust badge text | `#ffffff` | "ban" badge text -- white on amber |

**Accent reserved for:** Today date number circle, active view tab, solar term text tint, festival text. Same elements as Phase 2 plus lunar calendar elements.

**Fixed semantic colors:** `#ef4444` (holiday red) and `#f59e0b` (workday amber) are intentionally outside the Obsidian CSS variable system. These are semantically fixed colors representing China's statutory holiday/workday classification and should not shift with user theme accent changes. They are always mixed with `var(--background-primary)` via `color-mix()`, so they still adapt to light/dark themes.

**Today styling change (D-09):** Today cell removes its background tint (`color-mix(in srgb, var(--interactive-accent) 12%, ...)` is removed). Today is identified solely by the day number circle: `background: var(--interactive-accent); color: var(--text-on-accent); border-radius: 50%; width: 24px; height: 24px; font-weight: 600;` (already exists). This frees the cell background for holiday/workday indicators without visual conflict.

**Dark mode compatibility:** All `color-mix()` values use `var(--background-primary)` as the base, so tints adapt to dark themes automatically. Badge backgrounds use fixed hex mixed with theme background, producing appropriate muted tones in dark mode.

### New Visual Elements

#### Lunar Date Text (Month View)

Renders inline to the right of the Gregorian day number within each month day cell.

```
+------------------------------------------+
| 15 清明                                   |   <-- Day number + solar term (colored)
|  [10:00 Team standup         ]           |
|  [14:00 Design review        ]           |
+------------------------------------------+
```

- Position: Inline after `span.uni-calendar-day-number`, same flex row
- CSS class: `uni-calendar-lunar-text`
- Font: `var(--font-ui-smaller)`, weight 400, color `var(--text-faint)`
- Margin: `margin-left: 4px`
- Overflow: `text-overflow: ellipsis; white-space: nowrap; overflow: hidden;`
- Display priority (D-02): Traditional festival > Solar term > Lunar day
  - Festival: weight 600, color `var(--text-accent)`
  - Solar term: weight 400, color `color-mix(in srgb, var(--interactive-accent) 80%, var(--text-normal))`
  - Lunar day: weight 400, color `var(--text-faint)`

#### Day Number Row Layout

The day number and lunar text share a horizontal row at the top of each cell:

```css
.uni-calendar-day-top {
  display: flex;
  align-items: center;
  gap: 4px;
  min-height: 24px;
}
```

The existing `span.uni-calendar-day-number` and new `span.uni-calendar-lunar-text` are children of this wrapper.

#### Holiday Corner Badge

Small label in the top-right corner of day cells indicating statutory holiday ("xiu") or adjusted workday ("ban").

```
+------------------------------------------+
| 1 国庆                            [休]   |
|  [Events...]                             |
+------------------------------------------+
```

- CSS class: `uni-calendar-holiday-badge`
- Position: `position: absolute; top: 0; right: 4px;`
- The day cell (`.uni-calendar-day`) must have `position: relative;`
- Font: `var(--font-ui-smaller)`, weight 600, line-height 1.0
- Padding: `0 4px`
- Border-radius: `var(--radius-s)` (3px)
- "xiu" variant (`uni-calendar-holiday-badge--rest`):
  - Background: `color-mix(in srgb, #ef4444 75%, var(--background-primary))`
  - Color: `#ffffff`
  - Content: "休"
- "ban" variant (`uni-calendar-holiday-badge--work`):
  - Background: `color-mix(in srgb, #f59e0b 75%, var(--background-primary))`
  - Color: `#ffffff`
  - Content: "班"

#### Holiday Cell Background

Day cells on statutory holidays and adjusted workdays receive subtle background tints:

- Holiday rest: `background: color-mix(in srgb, #ef4444 8%, var(--background-primary))`
- Adjusted workday: `background: color-mix(in srgb, #f59e0b 8%, var(--background-primary))`
- CSS classes: `uni-calendar-day--holiday-rest`, `uni-calendar-day--holiday-work`
- Applies to all three views:
  - Month view: `.uni-calendar-day` element
  - Week view: `.uni-calendar-day-column` element
  - Day view: `.uni-calendar-day-body` column element
- Priority: Holiday background overrides the default `var(--background-primary)`. Today's cell no longer has its own background (D-09), so no conflict.
- Outside month cells (`.uni-calendar-day-outside`) with holidays: Use the holiday tint mixed with `var(--background-secondary-alt)` instead of `var(--background-primary)` to maintain the dimmed appearance.

#### Lunar Month in Toolbar Title

The toolbar title gains an appended lunar month name:

```
2026年4月 三月                       [< 今天 >] [月 周 日]
```

- Appended as a separate `span` after the existing title text
- CSS class: `uni-calendar-lunar-month`
- Font: `var(--font-ui-small)`, weight 400, color `var(--text-muted)`
- Margin: `margin-left: 8px`
- The lunar month shown corresponds to the 15th of the displayed Gregorian month (to handle months that span two lunar months -- pick the one covering the majority)

#### Week/Day View Holiday Indicators

Week view day headers and day view header gain holiday badges:

- Week view: Badge appears inside `.uni-calendar-week-day-header` element, after the day label text
- Day view: Badge appears inside `.uni-calendar-day-view-header` element
- Same badge styling as month view: `font-size: var(--font-ui-smaller); padding: 0 4px;`
- Background tint applied to day columns (week view) and day body (day view)

## Interaction Contract

### Lunar Date Display (D-01, D-02, D-03, D-04)

| Interaction | Behavior |
|-------------|----------|
| Month view day cell | Shows lunar text inline right of day number. Priority: festival > solar term > lunar day. |
| Month view title | Appends lunar month name after Gregorian year-month. |
| Week view header | No lunar date text. Holiday badge only if applicable. |
| Day view header | No lunar date text. Holiday badge only if applicable. |
| Lunar feature toggle off | All lunar text, holiday backgrounds, and badges hidden. Toolbar title reverts to Gregorian only. |

### Holiday Indicators (D-06, D-07, D-08)

| Interaction | Behavior |
|-------------|----------|
| Statutory holiday (xiu) | Cell background tinted warm red. "xiu" badge in top-right corner. Applies in month/week/day views. |
| Adjusted workday (ban) | Cell background tinted warm amber. "ban" badge in top-right corner. Applies in month/week/day views. |
| Regular day | No background tint, no badge. |
| Today on a holiday | Day number shows accent circle (today indicator). Cell background shows holiday tint. No conflict since today no longer uses background tint (D-09). |

### Settings Toggle (D-12)

| Setting | Default | Behavior |
|---------|---------|----------|
| showLunarCalendar | `true` | Toggles lunar date text, solar term coloring, festival names, lunar month in title |
| showHolidays | `true` | Toggles holiday/workday backgrounds and badges |

Both toggles are independent. User can show lunar dates without holidays, or vice versa.

## State Mapping

| UI State | Visual Treatment |
|----------|-----------------|
| Lunar enabled, holiday enabled | Full lunar overlay: dates, festivals, solar terms, holiday backgrounds, badges |
| Lunar enabled, holiday disabled | Lunar dates, festivals, solar terms shown. No holiday backgrounds or badges. |
| Lunar disabled, holiday enabled | No lunar text. Holiday backgrounds and badges shown. |
| Both disabled | Calendar appears identical to pre-Phase-6 state. |
| Holiday data loading | No badges or backgrounds until data arrives. Lunar dates render immediately (computed locally). |
| Holiday data fetch failed (online) | Show Notice: "节假日数据获取失败，将使用缓存数据". Use cached data if available. No badges if no cache. |
| Holiday data fetch failed (offline) | Use cached data silently. If no cache, no holiday indicators shown. |
| Year boundary | Holiday data fetched per year. When navigating to a new year, fetch that year's data. Cache per-year. |

## Architecture Notes for Implementer

### Month View Day Cell Structure (Modified)

```
.uni-calendar-day  (position: relative; for badge positioning)
  .uni-calendar-day-top  (new flex row wrapper)
    span.uni-calendar-day-number  (existing, possibly with today circle)
    span.uni-calendar-lunar-text  (new -- lunar day / festival / solar term)
  .uni-calendar-holiday-badge  (new, absolute positioned, conditional)
  div.uni-calendar-day-events  (existing -- event bars)
```

### CSS Class Application Logic

```
// Pseudocode for day cell class computation
let cls = 'uni-calendar-day';
if (isOutside) cls += ' uni-calendar-day-outside';
if (isToday) cls += ' uni-calendar-day-today';  // only affects day number, not background
if (isHolidayRest) cls += ' uni-calendar-day--holiday-rest';
if (isWorkdayAdjust) cls += ' uni-calendar-day--holiday-work';
```

### Today Styling Change (D-09)

Remove from CSS:
```css
/* REMOVE this rule */
.uni-calendar-day-today {
  background: color-mix(in srgb, var(--interactive-accent) 12%, var(--background-primary));
}
```

Keep:
```css
/* KEEP -- today is identified by the number circle only */
.uni-calendar-day-today .uni-calendar-day-number {
  background: var(--interactive-accent);
  color: var(--text-on-accent);
  border-radius: 50%;
  width: 24px;
  height: 24px;
  display: inline-flex;
  align-items: center;
  justify-content: center;
  font-weight: 600;
}
```

Also update week/day view today column:
```css
/* REMOVE background tint from today column */
.uni-calendar-day-column.is-today {
  /* Remove: background: color-mix(...) */
  /* Keep the class for potential future use, but no background */
  transition: background 0.3s ease;
}
```

### Lunar Data Module

New module: `src/lunar/LunarService.ts`

Responsibilities:
- Wrap third-party lunar library (selected during research phase)
- Expose: `getLunarDate(year, month, day)` returning `{ lunarDay: string, festival?: string, solarTerm?: string }`
- Expose: `getLunarMonth(year, month)` returning lunar month name for toolbar title
- Pure computation, no side effects, no async

### Holiday Data Module

New module: `src/lunar/HolidayService.ts`

Responsibilities:
- Fetch current-year mainland China holiday data from online API
- Cache in Obsidian data storage (per-year)
- Expose: `getHolidayInfo(dateStr)` returning `{ type: 'rest' | 'work' | null }`
- Handle offline gracefully (return cached data or null)

### Settings Extension

Add to `UniCalendarSettings` interface:
```typescript
showLunarCalendar: boolean;  // default: true
showHolidays: boolean;       // default: true
```
