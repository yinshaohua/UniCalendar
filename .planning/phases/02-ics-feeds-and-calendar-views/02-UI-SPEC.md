# Phase 2 — UI Design Contract

> Visual and interaction contract for ICS Feeds and Calendar Views. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Obsidian plugin -- no React/shadcn) |
| Preset | not applicable |
| Component library | Obsidian API (Modal, ItemView, Notice, Setting, setIcon) |
| Icon library | Lucide (bundled with Obsidian, accessed via `setIcon()`) |
| Font | `var(--font-interface)` (Obsidian system font stack) |

**Note:** All styling relies on Obsidian CSS custom properties. No external design system. Colors, spacing, and typography inherit from the user's Obsidian theme. Values below are expressed as Obsidian CSS variable references or explicit pixel values where Obsidian variables do not cover the need.

---

## Spacing Scale

Declared values (Obsidian `--size-4-*` tokens map to 4px increments):

| Token | Value | Obsidian Variable | Usage |
|-------|-------|-------------------|-------|
| xs | 4px | `var(--size-4-1)` | Icon-to-text gap, event bar inline padding |
| sm | 8px | `var(--size-4-2)` | Event bar vertical margin, compact element gaps |
| md | 12px | `var(--size-4-3)` | View padding, toolbar margin-bottom |
| lg | 16px | `var(--size-4-4)` | Modal section spacing, settings section gaps |
| xl | 24px | `var(--size-4-6)` | Modal body padding |
| 2xl | 32px | n/a (use `24px + 8px` or literal) | Not used in this phase |

Exceptions:
- Month day cells: `min-height: 70px` (already established in Phase 1)
- Week/Day hour cells: `min-height: 40px` week, `min-height: 48px` day (already established)
- Event block minimum height: `20px` (ensures tap target meets 44px when combined with padding on mobile -- see touch target note below)
- Mobile touch targets: All clickable event blocks must have at least `min-height: 32px` with `padding: 4px 8px`, yielding 40px effective touch area. Month view event bars use `min-height: 20px` with `padding: 2px 6px` (compact mode).

## Typography

All sizes reference Obsidian CSS variables for theme consistency.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | `var(--font-ui-small)` (~13px) | 400 | 1.4 | Event bar text in month view, modal body text |
| Label | `var(--font-ui-smaller)` (~11px) | 500 | 1.3 | Hour labels, overflow "+N more" indicator, event time range |
| Heading | `1.1em` (~15px) | 700 | 1.2 | Toolbar month/date label (already established) |
| Modal Title | `var(--font-ui-medium)` (~16px) | 600 | 1.3 | Event detail modal title |

**Weights used:** 400 (normal body/description), 600 (semibold for modal title, event title in week/day blocks)

---

## Color

This plugin inherits all surface and text colors from the active Obsidian theme. The color contract below maps to Obsidian CSS variables, not fixed hex values.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background-primary)` | Grid cells, modal background, view background |
| Secondary (30%) | `var(--background-secondary)` | Day headers, hour label column, view tab background, modal field labels |
| Accent (10%) | `var(--interactive-accent)` | Today highlight, active view tab, today's date circle |
| Destructive | `var(--text-error)` | Sync error state text only |

**Accent reserved for:** Today's date number circle background, today's column tint (`color-mix(in srgb, var(--interactive-accent) 12%, ...)`), active view tab background. Nothing else.

**Source colors (event color-coding):** Each calendar source uses its assigned color from `SOURCE_COLORS` array or user-overridden hex. Applied as:
- Month view: `border-left: 3px solid {sourceColor}` on event bars
- Week/Day view: `background: color-mix(in srgb, {sourceColor} 18%, var(--background-primary))` with `border-left: 3px solid {sourceColor}` on event blocks
- Event detail modal: Small `12px x 12px` color circle next to source name

## Component Inventory

### New Components (Phase 2)

| Component | Location | Description |
|-----------|----------|-------------|
| `EventBar` | Inline in CalendarView | Month view event snippet: colored left border + time + title |
| `EventBlock` | Inline in CalendarView | Week/Day view positioned block: colored background + border + title + time |
| `OverflowIndicator` | Inline in CalendarView | Month view "+N more" text, clickable |
| `EventDetailModal` | `src/views/EventDetailModal.ts` | Obsidian Modal subclass showing full event details |
| `IcsSyncAdapter` | `src/sync/IcsSyncAdapter.ts` | Not a UI component -- ICS fetch/parse (no visual contract) |

### EventBar (Month View Event Snippet)

```
+---+---------------------------+
| | 10:00 Team standup        |   <-- 3px colored left border, source color
+---+---------------------------+
```

- Container: `display: flex; align-items: center;`
- Left border: `border-left: 3px solid {sourceColor}; border-radius: 2px;`
- Padding: `2px 6px`
- Background: `var(--background-secondary)`
- Font: `var(--font-ui-smaller)`, weight 400, color `var(--text-normal)`
- Time prefix: `var(--font-ui-smaller)`, weight 500, color `var(--text-muted)`, margin-right `4px`
- Text overflow: `overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`
- Margin between bars: `2px 0`
- Cursor: `pointer`
- Hover: `background: var(--background-modifier-hover)`
- All-day events: No time prefix, just title. Placed before timed events.

### EventBlock (Week/Day View Positioned Block)

```
+---+---------------------------+
|   | Team standup              |
|   | 10:00-11:30               |   <-- Positioned absolutely on time grid
+---+---------------------------+
```

- Position: `absolute` within a `position: relative` day column container
- Top: calculated from `(startHour - gridStartHour) * hourHeight + (startMinute / 60) * hourHeight`
- Height: calculated from `durationMinutes / 60 * hourHeight`
- Min-height: `20px`
- Left border: `border-left: 3px solid {sourceColor}`
- Background: `color-mix(in srgb, {sourceColor} 18%, var(--background-primary))`
- Border-radius: `4px` (var(--radius-s))
- Padding: `4px 8px`
- Title: `var(--font-ui-small)`, weight 600, color `var(--text-normal)`, single line ellipsis
- Time: `var(--font-ui-smaller)`, weight 400, color `var(--text-muted)`
- Cursor: `pointer`
- Hover: increase background opacity to `color-mix(in srgb, {sourceColor} 28%, var(--background-primary))`
- Overlapping events: side-by-side columns. 2 overlapping = each 50% width. 3 = each 33%. Calculated via overlap detection algorithm.

### OverflowIndicator (Month View "+N more")

- Text: `+{N} more` (Chinese: `+{N} 更多`)
- Font: `var(--font-ui-smaller)`, weight 500, color `var(--text-accent)`
- Padding: `2px 6px`
- Cursor: `pointer`
- Hover: `text-decoration: underline`
- Click action: Switch to day view for that date (Mode 2 only)

### EventDetailModal

Uses Obsidian `Modal` base class. Layout:

```
+--------------------------------------+
|  [X]  Event Title                    |   <-- Modal header
+--------------------------------------+
|                                      |
|  [clock icon] Wed, Apr 1 10:00-11:30 |   <-- Start/end with icon
|                                      |
|  [map-pin icon] Meeting Room 3A      |   <-- Location (if present)
|                                      |
|  [align-left icon] Description text  |   <-- Description (if present)
|  that can span multiple lines...     |
|                                      |
|  [circle] Work Calendar              |   <-- Source name with color dot
|                                      |
+--------------------------------------+
```

- Modal width: Obsidian default modal width (approx 480px desktop, full-width mobile)
- Title: `var(--font-ui-medium)`, weight 600, color `var(--text-normal)`, `margin-bottom: 16px`
- Field rows: Each field is a flex row with icon + text
  - Icon: Lucide icon via `setIcon()`, size 16px, color `var(--text-muted)`, `margin-right: 8px`
  - Text: `var(--font-ui-small)`, weight 400, color `var(--text-normal)`
- Field spacing: `margin-bottom: 12px` between each field row
- Source indicator: `12px x 12px` circle (`border-radius: 50%`) filled with source color, inline with source name
- Description: `white-space: pre-wrap` to preserve line breaks from ICS, `max-height: 200px; overflow-y: auto` if long
- Icons used: `clock` (time), `map-pin` (location), `align-left` (description), none (source -- uses color circle)

## Interaction Contract

### Month View Event Rendering

| Interaction | Behavior |
|-------------|----------|
| Display events | Render EventBars in each day cell, ordered by start time. All-day events first, then timed events. |
| Overflow Mode 1 (default) | Show all events. Day cell height auto-expands with content. |
| Overflow Mode 2 | Show first 3 events + OverflowIndicator. Click indicator switches to day view for that date. |
| Click event bar | Open EventDetailModal with full event details. |
| Click day number | Switch to day view for that date. |

### Week/Day View Event Rendering

| Interaction | Behavior |
|-------------|----------|
| Display events | Position EventBlocks absolutely within day column containers based on start time and duration. |
| Overlap handling | Detect overlapping time ranges. Split into equal-width columns. Max 4 columns (events beyond 4 overlap stack in 4th column). |
| Click event block | Open EventDetailModal with full event details. |
| Current time indicator | Red horizontal line at current time position in today's column. `border-top: 2px solid var(--text-error); position: absolute; width: 100%; z-index: 5;` with a small `6px` red circle at the left edge. |

### Week View Hour Range

| Condition | Range |
|-----------|-------|
| Default | 7:00 -- 22:00 (15 hours) |
| Events outside range | Extend range to include earliest/latest event, rounded down/up to whole hour |
| Day view | Always 0:00 -- 23:00 (full 24 hours, already implemented) |

### Navigation and Keyboard (VIEW-06)

| Key | Month View | Week View | Day View |
|-----|-----------|-----------|----------|
| Left Arrow | Previous month | Previous week | Previous day |
| Right Arrow | Next month | Next week | Next day |
| `t` | Navigate to today | Navigate to today | Navigate to today |
| `1` | Switch to month | Switch to month | Switch to month |
| `2` | Switch to week | Switch to week | Switch to week |
| `3` | Switch to day | Switch to day | Switch to day |

Keyboard listeners: Attach to the CalendarView container element. Only active when the calendar view has focus. Use `tabindex="0"` on the content container to make it focusable.

### ICS Sync Flow

| State | Visual Feedback |
|-------|----------------|
| Sync triggered | Sync status shows "同步中..." with accent color (already implemented) |
| Sync success | Status updates to "刚刚同步", calendar re-renders with new events |
| Sync error (network) | Status shows "同步出错" in error color. Click expands: "无法连接到日历源: {url}. 请检查网络连接和URL是否正确." |
| Sync error (parse) | Status shows "同步出错" in error color. Click expands: "日历数据解析失败: {url}. 该链接可能不是有效的ICS订阅源." |
| No sources configured | Status shows "未配置日历源" (already implemented) |

## State Mapping

| UI State | Visual Treatment |
|----------|-----------------|
| Loading (first sync) | Existing grid renders immediately. Sync status shows "同步中...". Events populate once sync completes. No skeleton -- grid is already visible. |
| Events loaded | EventBars/EventBlocks rendered in cells. Source colors applied. |
| No events for visible range | Grid renders normally with no event elements. No special empty state (the grid itself is the content). |
| Sync error | Sync status indicator turns error color. Events from cache (if any) remain visible. |
| Offline with cache | Cached events display normally. Sync button click shows Notice: "无法同步, 请检查网络连接." |
| Recurring event | Expanded instances rendered identically to single events. No visual distinction for recurring vs. single. |

## Architecture Notes for Implementer

### Week/Day View Column Layout

The current week/day grids use CSS Grid with fixed cells per hour. To support absolute positioning of event blocks:

1. Each day column in week view needs a `position: relative` wrapper that spans the full hour range height
2. Event blocks are `position: absolute` children of this wrapper
3. Hour height constant: `40px` (week), `48px` (day) -- matches existing `min-height` values
4. Event top = `(startHour - firstHour) * hourHeight + (startMinutes / 60) * hourHeight`
5. Event height = `max(20, durationMinutes / 60 * hourHeight)`

This requires restructuring the week/day grid from pure CSS Grid cells to a column-based layout with absolute positioning inside each column. The hour labels column remains as-is on the left.

### Overlap Detection Algorithm

For each day column:
1. Sort events by start time
2. Walk through events, tracking active "columns" (time slots occupied)
3. When an event overlaps existing columns, assign it to the next available column
4. After assignment, set `width: 100% / totalColumns` and `left: columnIndex * (100% / totalColumns)`
5. Cap at 4 visual columns maximum

### Month View Cell Structure

Each month day cell (`.uni-calendar-day`) gains an events container:
```
.uni-calendar-day
  span.uni-calendar-day-number  (existing)
  div.uni-calendar-day-events   (new -- contains EventBars)
    div.uni-calendar-event-bar  (per event)
    div.uni-calendar-overflow   ("+N more" if Mode 2)
```
