**3. Add palette swatches in AddSourceModal (after the color picker Setting):**

After the color picker Setting block (lines 305-310), add a palette swatch row. Replace the simple `addColorPicker` setting with a combined approach:

Find the color Setting in `showSourceForm()` (the `new Setting(contentEl).setName('颜色')...addColorPicker(...)` block around lines 305-310). After this Setting, add:

```typescript
const paletteRow = contentEl.createDiv({ cls: 'uni-calendar-palette-row' });
for (const { name: paletteName, hex } of RECOMMENDED_PALETTE) {
  const swatch = paletteRow.createDiv({ cls: 'uni-calendar-palette-swatch' });
  swatch.style.background = hex;
  swatch.setAttribute('aria-label', paletteName);
  swatch.setAttribute('title', paletteName);
  if (hex === color) swatch.addClass('is-selected');
  swatch.addEventListener('click', () => {
    color = hex;
    // Update the color picker if it exists
    const pickerEl = contentEl.querySelector<HTMLInputElement>('input[type="color"]');
    if (pickerEl) {
      pickerEl.value = hex;
      pickerEl.dispatchEvent(new Event('input'));
    }
    // Update selected state on swatches
    paletteRow.querySelectorAll('.uni-calendar-palette-swatch').forEach(s => s.removeClass('is-selected'));
    swatch.addClass('is-selected');
  });
}
```

Actually, check: the modals render inside Obsidian's modal container which shares the document. Since CARD_STYLES is injected into the settings container and modals are in the same document, the CSS applies globally. BUT if the user opens a modal before the settings tab renders (unlikely but possible), the styles won't exist.
