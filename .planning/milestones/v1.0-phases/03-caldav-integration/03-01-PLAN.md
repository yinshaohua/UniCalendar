    **Export: `DiscoveredCalendar` interface:**
    ```typescript
    export interface DiscoveredCalendar {
      href: string;       // Calendar path e.g. "/calendars/user/personal/"
      displayName: string; // Human-readable name
    }
    ```

    **Export: `CalDavSyncAdapter` class with these methods:**

    1. `constructor(icsAdapter: IcsSyncAdapter)` — Takes IcsSyncAdapter for reusing parseIcsText.

    2. `async discoverCalendars(serverUrl: string, username: string, password: string): Promise<DiscoveredCalendar[]>` — Per D-03, performs 3-step PROPFIND chain:
       - Step 1: PROPFIND `${serverUrl}/.well-known/caldav/` with Depth: 0 to get `current-user-principal` href. If 404, fallback: PROPFIND on `${serverUrl}/` directly (per D-02 DingTalk may not support .well-known).
       - Step 2: PROPFIND on principal URL with Depth: 0 to get `calendar-home-set` href.
       - Step 3: PROPFIND on calendar-home-set URL with Depth: 1 to list all collections. Filter to those where `<resourcetype>` contains an element with `localName === 'calendar'`. Return array of `{ href, displayName }`.
       - All requests use Basic Auth header: `Authorization: Basic ${btoa(username + ':' + password)}`
       - Use `requestUrl` from obsidian for all HTTP calls. Set method to 'PROPFIND'.
       - Parse XML responses with `new DOMParser().parseFromString(text, 'text/xml')`.
       - Resolve relative hrefs against serverUrl origin.
       - XML request bodies per 03-RESEARCH.md.

    3. `async sync(source: CalendarSource, rangeStart: Date, rangeEnd: Date): Promise<CalendarEvent[]>` — matching IcsSyncAdapter pattern:
       - Validate `source.caldav` exists, throw "日历源缺少CalDAV配置" if not.
       - Validate `source.caldav.calendarPath` exists, throw "日历源缺少CalDAV日历路径, 请先发现日历" if not.
       - Build calendar URL from `serverUrl` + `calendarPath`.
       - Send REPORT request (method: 'REPORT') with `calendar-query` XML body from 03-RESEARCH.md, using `dateToCalDavUTC(rangeStart)` and `dateToCalDavUTC(rangeEnd)` for time-range filter.
       - Parse 207 multi-status response: extract `<calendar-data>` text content from each `<response>`.
       - For each calendar-data block, call `this.icsAdapter.parseIcsText(icsText, source.id, rangeStart, rangeEnd)`.
       - Concatenate all events and return.
       - On 401 response: throw "CalDAV认证失败: 请检查用户名和密码".
       - On other HTTP errors: throw "CalDAV同步失败: " + status code + statusText.

    4. Private helper `dateToCalDavUTC(d: Date): string` — Converts Date to `YYYYMMDDTHHmmssZ` format: `d.toISOString().replace(/[-:]/g, '').split('.')[0] + 'Z'`.

    5. Private helper `createBasicAuthHeader(username: string, password: string): string` — Returns `'Basic ' + btoa(username + ':' + password)`.

    6. Private helpers for XML parsing:
       - `parseMultiStatusXml(xmlText: string)`: Parse with DOMParser, iterate `<response>` elements, extract `href` and properties. Use `getElementsByTagNameNS('DAV:', 'response')` or iterate and check `localName`.
       - `parseEventReportXml(xmlText: string): string[]`: Extract all `<calendar-data>` text contents.

    **Important implementation notes:**
    - Use `localName` property for XML element matching (avoids namespace prefix issues per 03-RESEARCH.md risk mitigation).
    - requestUrl calls: `requestUrl({ url, method: 'PROPFIND'|'REPORT', headers: { Authorization, 'Content-Type': 'application/xml', Depth: '0'|'1' }, body: xmlBody })`.
    - For URL resolution: use `new URL(href, serverUrl).href` to resolve relative paths.

Test discoverCalendars with mocked 3-step chain. Test sync with mocked REPORT response. Test error cases (401, missing config).
