    Create `src/sync/GoogleAuthHelper.ts` with class `GoogleAuthHelper`:

    - `REDIRECT_URI_DEFAULT = 'obsidian://uni-calendar/oauth-callback'` (per D-01 modified -- using obsidian:// protocol handler)
    - `TOKEN_ENDPOINT = 'https://oauth2.googleapis.com/token'`

    - `SCOPE = 'https://www.googleapis.com/auth/calendar.readonly'`
    - `TOKEN_REFRESH_BUFFER_MS = 5 * 60 * 1000` (5 minutes before expiry)

    Methods:
    - `generateCodeVerifier(): string` -- Uses `crypto.getRandomValues(new Uint8Array(32))`, base64url encodes
    - `async generateCodeChallenge(verifier: string): Promise<string>` -- SHA-256 hash via `crypto.subtle.digest`, base64url encode
    - `private base64UrlEncode(buffer: Uint8Array): string` -- btoa + replace +/- _/ remove =
    - `buildAuthUrl(clientId: string, redirectUri: string, codeVerifier: string, state: string): Promise<{ url: string; codeVerifier: string }>` -- Constructs Google OAuth2 URL with all params including PKCE code_challenge (S256)
    - `async exchangeCode(code: string, clientId: string, clientSecret: string, redirectUri: string, codeVerifier: string): Promise<{ accessToken: string; refreshToken: string; tokenExpiry: number }>` -- POST to token endpoint with `grant_type=authorization_code`, parses `access_token`, `refresh_token`, `expires_in` (convert to absolute timestamp: `Date.now() + expires_in * 1000`)
    - `async refreshAccessToken(refreshToken: string, clientId: string, clientSecret: string): Promise<{ accessToken: string; tokenExpiry: number }>` -- POST to token endpoint with `grant_type=refresh_token`. On error (status >= 400), throw Error with Chinese message "令牌刷新失败，请重新授权". IMPORTANT per Pitfall 4: does NOT return refreshToken (it stays the same)
    - `async ensureValidToken(google: CalendarSource['google']): Promise<string>` -- If `tokenExpiry` is more than 5 min in the future, return existing `accessToken`. Otherwise call `refreshAccessToken`, update the google object in-place (accessToken, tokenExpiry), return new token. Throw if no refreshToken exists.

    All HTTP calls use `requestUrl` from obsidian (per D-05). All error messages in Chinese (per project convention).
